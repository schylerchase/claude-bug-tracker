## Context

- Project root: !`git rev-parse --show-toplevel`
- Current branch: !`git branch --show-current`
- Arguments: $ARGUMENTS

## Your Task

You are the bug regression tracker. Parse the arguments and execute the matching subcommand.

### Subcommands

**`scan`** - Scan git history for `fix(` commits and build a fragile area map:

1. Run: `bash "${CLAUDE_PLUGIN_ROOT}/scripts/scan-commits.sh"`
2. Read the output (fragile areas with fix counts per file/scope)
3. Update or create `bugs.md` in the project root:
   - Preserve existing Active and Watch sections
   - Replace the "Fragile Areas" section with fresh data from the scan
   - Add timestamp to the section header
4. Display a summary table of fragile areas

**`list`** - Show current bug tracking state:

1. Read `bugs.md` from the project root
2. Display each section: Active bugs, Fragile Areas, Watch on Refactor
3. If `bugs.md` doesn't exist, say so and suggest running `/bug-track scan`

**`add <description>`** - Log a new bug:

1. Read `bugs.md` (create if missing with template)
2. Generate next BUG-NNN ID from existing entries
3. Add to Active section: `- [BUG-NNN] <description> (<date>)`
4. Write updated file
5. Confirm the addition

**`resolve <id>`** - Mark a bug as fixed:

1. Read `bugs.md`
2. Find the matching BUG-NNN entry in Active
3. Move it to a "Resolved" section with today's date
4. Write updated file
5. Confirm the resolution

### bugs.md Template

When creating a new bugs.md, use this structure:

```markdown
# Bug Tracking

## Active
<!-- [BUG-001] Description (date) -->

## Fragile Areas
<!-- Auto-generated by /bug-track scan -->

## Watch on Refactor
<!-- Patterns that break when modified -->

## Resolved
<!-- Moved from Active when fixed -->
```

### No Arguments

If no arguments provided, show available subcommands and a brief status (whether bugs.md exists, last scan date if available).
